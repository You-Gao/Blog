# [roadmap.sh](https://roadmap.sh/u/yougao)

I found this site a long time ago and regained interest in it because I wanted to benchmark my career. The best thing about this site is how software developers have somehow reached a consensus on the standard tooling and concepts.

If you were to use the site as a benchmark, do not lie to yourself. roadmap.sh exists for you to be truthful about where you stand currently. If you are on the borderline of "*oh, I know that*", ask yourself if you actually do. It is easy to say you know everything, but there is a "*tacit*" dimension to be recognized before marking it off.

## My Goals

I filled out my roadmaps and made .txt files regarding the main concepts for each career path, with specific concepts you should master. They are ordered based on how much I enjoy working in the specific role. A good ethos to go by is: "*don't check it off if you haven't implemented/applied the concepts somewhere.*"

## 10x, 100x, and the 1000x Unicorn

After working through the developer roadmaps, I realized there wasn't a focus on the broader skill sets a back-end or front-end SWE might need. So I used the AI function to create roadmaps for the illustrious 10-1000x programmers. 

Not surprisingly, the main topics were programming constructs, optimization, computer architecture, operating systems, algorithms, and foundational problem-solving skills.

These lists were mainly made for the fun of it, but they turned out to be solid in showing what is needed beyond knowing how to use specific tools and domain concepts.
